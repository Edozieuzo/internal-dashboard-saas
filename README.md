Internal Dashbord SaaS (v1)

This was my very first attempt at building a full stack project. It was built before I had stuctured frontend and backend knowledge.
The goal was simple:
1. Learn how the web works
2. Learn how frontend and backend connects
3. Try to build a real system beyound tutotials

Features
1. Basic user authetication (though incomplete)
2. Simple dashboard UI (no CSS)
3. Basic navigation between pages

Known Issues
This version had several limitations due to my early stage learning:
1. No backend structure (just JavaScript for web behaviour)
2. Authetication system is not secure or anywhere near production ready
3. No proper role based access control
4. Limited security considerations

What I learned from this
1. How authetication flows should work and were mine was weak
2. The importance of backend
3. The gap between building something that works vs something that is correct

Future Plans
This project will be rebuilt as internal-dashboard-saas-v2 after I complete structured learning through the odin project open soure platform
1. Clean backend structure
2. secure authentication system
3. Role based access control (adim/user seperation)
4. Input validation and security hardening
5. Improved UI structure and scalability

Conclusion
This project is archived as a learning milestone. Not production ready, not maintained. It represents the starting point of my full stack journey in understanding the web, where vunerabilities lie, how to break them and fix them. 
Every mistake in this project is intentional in hindsight, it shows where my understanding was at the time, and serves as a reference point for improvment in future versions.
